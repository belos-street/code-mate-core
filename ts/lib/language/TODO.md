下面是后续建议接入的热门语言（按优先级）。

## P0（优先，建议先做）

- [x] `typescript`
  - 说明：前端/Node 场景最常见，和现有 `javascript` 规则复用度高，投入小、收益大。

- [x] `html`
  - 说明：Web 展示必备语言，可与 JS/CSS 形成完整前端代码块高亮组合。

- [x] `css`
  - 说明：和 HTML 一起使用频率极高，语法相对规则，适合快速落地。

- [x] `bash`（shell）
  - 说明：工程文档、CI、运维脚本高频出现，规则简单，能快速提升通用性。

## P1（第二梯队，需求广）

- [x] `sql`
  - 说明：后端和数据场景非常常见，关键字与函数语义清晰，适合做标准化高亮。

- [x] `yaml`
  - 说明：K8s、GitHub Actions、配置文件大量使用；和 JSON 一起覆盖配置生态。

- [x] `markdown`
  - 说明：文档与 AI 场景核心格式，代码块/标题/列表/强调语法实用价值高。

- [x] `java`
  - 说明：企业后端常用，生态大，落地后可覆盖很多服务端仓库。

- [x] `c`
  - 说明：系统与底层项目常见，生态存量大；先做注释/关键字/字符串/数字/预处理指令的实用高亮。

- [x] `cpp`（C++）
  - 说明：工程与性能场景广泛使用；首版优先覆盖常见语法，模板元编程等复杂结构可后置。

## P2（按业务再上）

- [x] `go`
  - 说明：云原生与基础设施项目常见，语法规则相对稳定，维护成本可控。

- [x] `rust`
  - 说明：系统/高性能方向热门，语法有一定复杂度，但社区增长快。

- [x] `csharp`
  - 说明：企业与游戏（Unity）场景常见，需求稳定。

- [x] `php`
  - 说明：存量项目多，尤其 Web 后端维护场景价值高。

## 复杂语法降级策略（强烈建议）

- 对复杂且低频的语法片段，允许直接降级为 `default` 文本 token。
- 优先保证：
  - 不崩溃
  - 行列追踪正确
  - 常见语法高亮稳定
- 典型可降级场景：
  - C/C++ 宏技巧、复杂模板、预处理嵌套边界
  - 各语言非常规/低频写法
- 原则：先覆盖 80% 常见代码，再逐步补齐边角语法。
